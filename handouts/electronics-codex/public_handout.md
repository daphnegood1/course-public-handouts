# 大學生電子學初學者的 Codex 入門公開講義

## 這份講義要解決的問題

電子學初學者常遇到三個困難：

1. 公式看得懂，但不知道怎麼整理成可操作的解題流程。
2. 電路圖、量測資料、實驗紀錄分散在不同檔案，很難回頭追蹤。
3. 想用 AI 幫忙，但常常只得到一段答案，沒有形成可檢查、可修改、可累積的學習系統。

Codex 的價值不是代替你學電子學，而是幫你把「讀題、整理、推導、驗算、做筆記、建立小工具」變成一個可重複的工作流程。

## 學習目標

完成這份講義後，你應該能夠：

- 建立一個安全的 Codex 專案工作區。
- 用 Codex 整理電子學課堂筆記、實驗紀錄與解題步驟。
- 用清楚的提示詞請 Codex 協助分析電路題。
- 建立 GitHub 版本紀錄，保留每次修改過程。
- 用 Obsidian 管理電子學學習筆記。
- 理解什麼內容可以交給 AI，什麼內容仍需要自己驗算與判斷。

## 課程路線

| 階段 | 主題 | 電子學學習情境 |
|---|---|---|
| 1 | 安裝與安全設定 | 建立 `electronics-lab-notes` 專案 |
| 2 | Plugins 與 Skills | 讓 Codex 能處理文件、簡報、網頁與圖表 |
| 3 | GitHub | 追蹤解題筆記與小工具版本 |
| 4 | Obsidian | 建立電子學第二大腦 |
| 5 | Firebase / 互動工具 | 製作班級共用的電子學練習工具 |
| 6 | 專案工作流 | 用初始化、開工、收工維持學習連續性 |

## 1. 建立 Codex 專案工作區

先建立一個專門放電子學學習資料的資料夾，例如：

```text
electronics-codex-lab/
├─ notes/
├─ exercises/
├─ lab-data/
├─ tools/
└─ README.md
```

這個資料夾就是 Codex 的工作桌。所有 AI 產出的筆記、練習整理、小工具與檢查紀錄都放在這裡。

### 建議一開始建立的檔案

| 檔案 | 用途 |
|---|---|
| `README.md` | 說明這個電子學學習專案要做什麼 |
| `AGENTS.md` | 告訴 Codex 工作規則，例如不要直接給答案，要先列推導 |
| `.gitignore` | 避免把暫存檔、密鑰或大型輸出放進 Git |

## 2. 用 Codex 整理電子學題目

不要只問：

```text
幫我解這題。
```

比較好的問法是：

```text
請用大一電子學初學者能理解的方式，協助我分析這題二極體電路。

請按照以下格式：
1. 先列出已知條件。
2. 判斷二極體可能的導通狀態。
3. 寫出假設。
4. 用 KVL 或 KCL 推導。
5. 檢查假設是否成立。
6. 最後給我一個可複習的解題流程。

請不要只給答案。
```

這樣 Codex 比較容易產出「可學習」的解題過程，而不是只給結果。

## 3. 電子學筆記的標準格式

建議每一個主題都整理成同一種格式。

```markdown
# 主題：二極體導通判斷

## 觀念

## 常見題型

## 解題流程

## 易錯點

## 例題

## 我自己的理解
```

當格式固定，你之後要整理 BJT、MOSFET、OP Amp、RC 暫態響應時，就能快速累積成一套完整學習庫。

## 4. GitHub：保留你的學習修改紀錄

GitHub 對電子學學習的用途不是只有寫程式。它可以幫你保存：

- 每次修改後的解題筆記。
- 電路計算小工具。
- 實驗資料整理程式。
- 期中、期末複習版筆記。
- 小組專題的版本紀錄。

例如你建立一個分壓器計算工具：

```text
tools/voltage-divider.html
```

每次你改公式、改介面、加說明，都可以用 GitHub 追蹤。這樣出錯時能回到上一版，也能和同學合作。

## 5. Obsidian：建立電子學第二大腦

Obsidian 適合放長期筆記。你可以用以下分類：

```text
電子學/
├─ 00_課程索引.md
├─ 01_二極體.md
├─ 02_BJT.md
├─ 03_MOSFET.md
├─ 04_OP_Amp.md
├─ 05_實驗紀錄/
└─ 06_錯題整理/
```

Codex 可以協助你把課堂內容整理成 Obsidian 筆記，但你仍要檢查公式、符號方向與假設是否正確。

## 6. 用 Codex 建立電子學小工具

適合初學者練習的小工具：

| 小工具 | 學習目的 |
|---|---|
| 分壓器計算器 | 練習歐姆定律與串聯電阻 |
| RC 時間常數視覺化 | 理解充放電曲線 |
| 二極體導通判斷流程圖 | 練習分段線性模型 |
| BJT 偏壓檢查表 | 練習工作點判斷 |
| OP Amp 反相 / 非反相計算器 | 練習理想放大器假設 |

範例提示詞：

```text
請幫我建立一個單頁 HTML 小工具，主題是 RC 充電曲線。

使用者可以輸入 R、C、Vin，頁面要顯示：
1. 時間常數 tau = R*C。
2. 5 tau 後接近穩態的說明。
3. 簡單表格列出 0 tau、1 tau、2 tau、3 tau、5 tau 的電壓。

請用大學生初學電子學能看懂的文字說明。
```

## 7. 專案工作流：初始化、開工、收工

### 初始化

第一次建立專案時，請 Codex 建立資料夾、README、規則與學習目標。

```text
請幫我初始化一個電子學學習專案。
目標是整理本學期的二極體、BJT、MOSFET、OP Amp 筆記與練習題。
```

### 開工

每次重新開始時，先讓 Codex 讀取目前狀態。

```text
請讀取這個電子學學習專案目前的 README 和最近筆記，告訴我今天可以接續哪三件事。
```

### 收工

結束前，請 Codex 寫下完成事項、卡住的地方與下一步。

```text
請幫我收工：整理今天完成的內容、還沒解決的問題、下一次要做的三件事。
```

這三個動作能避免每次重新開對話都從零開始。

## 8. 公開版安全提醒

公開講義中可以放：

- 通用學習流程。
- 公開工具連結。
- 已整理過的範例題。
- 不含個資的操作步驟。

不應公開：

- 老師備課備註。
- 學生個資或作業檔案。
- 私有 Google Drive、Obsidian、GitHub 連結。
- 未授權逐字稿。
- API key、token、帳號資訊。

## 課堂練習

### 練習 1：建立電子學 Codex 專案

建立一個資料夾，命名為：

```text
electronics-codex-lab
```

請 Codex 幫你建立：

- `README.md`
- `notes/`
- `exercises/`
- `tools/`

### 練習 2：整理一題二極體電路

選一題課本中的二極體題目，請 Codex 按照「已知條件、假設、推導、驗算、結論」整理。

### 練習 3：做一個 RC 時間常數小工具

請 Codex 產出一個 HTML 小工具，能輸入 R、C、Vin 並顯示 tau 和電壓表。

## 總結

Codex 對電子學初學者最有價值的地方，不是直接給答案，而是幫你建立可重複的學習流程：

```text
整理題目
→ 推導步驟
→ 驗算假設
→ 做成筆記
→ 建立小工具
→ 用 GitHub 與 Obsidian 長期累積
```

只要你把每次學習都留下紀錄，Codex 就能從一次性的問答工具，變成你的電子學學習助教。
